# 项目全局规范-变更记录

## 2026-05-03

- 变更类型：新增
- 变更项：历史记录模式筛选约定
- 变更摘要：新增历史记录页必须展示每条记录所属的生图模式，并支持按模式筛选历史记录列表。
- 变更原因：历史记录已同时承载对话、分镜、生成和编辑等多种模式，不加模式筛选会降低定位效率，也与图库页现有筛选体验不统一。

- 变更类型：新增
- 变更项：普通生图与编辑模式会话化执行约定
- 变更摘要：新增普通生图模式和图片编辑模式也必须归属到各自独立会话；同一会话内允许连续提交多次请求，但每次都必须直接按用户当次完整提示词执行，禁止调用 planner 生成助手回复，也禁止自动扩写提示词后再生图。
- 变更原因：用户要求普通生图和编辑模式在保留会话归属的同时，执行链路必须保持“直接生图/直接编辑”，不能被对话式 AI 回复和提示词扩写污染。

- 变更类型：新增
- 变更项：失败上下文过滤与失败任务重试约定
- 变更摘要：新增对话/分镜失败后继续原会话时，planner 上下文只能包含用户消息、已完成助手回复和成功图片提示词，禁止带入失败原因、未完成占位内容和失败生图提示词；同时普通生图、编辑和分镜在失败或取消后必须支持基于原请求参数和原输入图重试，并继续遵守同一会话单任务约束。
- 变更原因：避免失败任务的错误信息和未完成内容污染后续对话上下文，同时补齐普通生图与编辑失败后无法原请求重试的问题。

- 变更类型：新增
- 变更项：设置分组显式保存反馈约定
- 变更摘要：新增设置面板中的每个独立设置分组都必须提供单独“保存修改”按钮，并在按钮上方原位展示成功或失败反馈，禁止继续依赖静默自动保存。
- 变更原因：用户需要明确知道当前修改是否已经真正落库，避免设置改动只在本地状态生效却没有被可靠持久化。

- 变更类型：新增
- 变更项：图库模式标识与筛选约定
- 变更摘要：新增图库页必须展示每个会话图片分组所属的生图模式，并支持按模式筛选分组结果。
- 变更原因：历史图片按会话聚合后，仍需要快速区分它们属于对话、分镜、生成还是编辑模式，降低查找成本。

- 变更类型：新增
- 变更项：继续改跨会话引用与底部参考图预览约定
- 变更摘要：新增从历史记录或图库点击“继续改”时，若目标图片不属于当前会话同模式的可复用生成图，前端必须将该图片转换为本轮上传参考图后再进入处理界面；同时工作台底部必须展示已选参考图的真实缩略图预览。
- 变更原因：避免跨会话继续改时仅保存图片 id 导致后端被当前会话过滤后失效，也避免用户只看到“已选”文字却看不到具体参考图内容。

- 变更类型：新增
- 变更项：项目统一密码页访问约定
- 变更摘要：新增项目访问必须先通过统一密码页验证；未完成验证前，禁止直接访问首页、前端静态资源、媒体资源和业务 API。
- 变更原因：项目已要求增加统一访问密码保护，需要确保所有入口都先经过同一套验证，而不是只拦首页不拦接口或静态资源。

- 变更类型：新增
- 变更项：图库按会话聚合展示约定
- 变更摘要：新增图库页必须按会话聚合同一会话的全部历史图片；当单个会话图片过多时，默认仅展示部分预览图，并提供“查看详情”入口展开全部图片；图库预览切换必须在当前会话图片集合内进行。
- 变更原因：保证历史图片浏览时不会把同一会话拆散到多个分组中，方便用户按会话连续查看和在预览弹窗中前后切换。

- 变更类型：新增
- 变更项：全局生图提供商池与单提供商并发约定
- 变更摘要：新增最终生图执行必须统一走全局生图提供商池；普通生图、编辑、对话、分镜与分镜重试共享同一池；默认每个 provider 最多同时运行 3 个生图任务；调度必须优先空闲 provider，其次选择运行中任务最少的 provider；任务与前端界面必须展示当前 provider 和池统计。
- 变更原因：项目已接入多个 GPT 提供商，需要在不突破单 provider 并发限制的前提下提升整体生图吞吐，并让用户可见当前 provider 使用情况与池容量。

- 变更类型：新增
- 变更项：会话模式隔离与运行锁定约定
- 变更摘要：新增对话模式与分镜模式必须使用各自独立会话；同一会话存在 `queued` 或 `running` 任务时禁止继续发送新消息；从历史记录继续对话时必须复用原 `conversation_id` 与原上下文。
- 变更原因：避免同一会话在任务运行期间继续追加消息导致上下文串扰，也避免聊天与分镜混用同一会话破坏历史语义和继续对话行为。

- 变更类型：新增
- 变更项：Git 提交信息中文化约定
- 变更摘要：新增 Git 提交信息必须使用中文；提交标题必须为中文短标题，提交正文必须使用中文详细描述本次改动的核心内容与目的。
- 变更原因：统一项目提交记录的阅读体验与追溯方式，降低后续排查和协作时的语言混杂成本。

- 变更类型：新增
- 变更项：分镜逐张图片实时展示约定
- 变更摘要：新增分镜连续生图每完成并保存一张镜头图片后，后端必须立即通过任务级 SSE 推送镜头状态和图片信息，前端必须立即展示该图片，禁止等全部镜头完成后才统一刷新。
- 变更原因：分镜任务耗时较长，用户需要按镜头顺序实时看到每张已完成图片和当前进度，便于确认连续性并降低等待不确定性。

## 2026-05-02

- 变更类型：新增
- 变更项：云服务器更新脚本数据库保护约定
- 变更摘要：新增项目更新脚本禁止覆盖、重置或重写云服务器已有 SQLite 业务数据；若版本涉及表结构变化，只允许执行向前兼容的 schema 迁移，例如创建缺失表或补充缺失列。
- 变更原因：云服务器数据库承载长期历史任务、会话、图片和 provider 配置，项目更新必须保护业务数据，同时保证代码升级后的表结构兼容。

- 变更类型：新增
- 变更项：AI planner 接口格式兼容约定
- 变更摘要：新增 AI planner 默认使用 `/v1/responses`，仅在用户显式选择时允许切换为 `/v1/chat/completions`；`chat/completions` 仅用于文本规划，不得影响 image_generation 生图执行链路。兼容模式只保证传入文字对话、参考图文字说明和已知生图提示词。
- 变更原因：兼容通义、DeepSeek 等只支持或更适合 `chat/completions` 的文本规划模型，同时确保现有 Responses 生图功能不受影响。

- 变更类型：新增
- 变更项：前端图片预览交互约定
- 变更摘要：新增图片预览必须使用站内固定大弹窗，支持原图展示、拖拽移动、缩放，以及同一任务、会话或图库分组内上一张/下一张切换，禁止默认打开新页面。
- 变更原因：提升历史图片、任务图片和对话图片的查看效率，避免预览图片时跳出当前工作流。

- 变更类型：新增
- 变更项：任务级 SSE 实时反馈约定
- 变更摘要：新增前端必须通过任务级 SSE 事件流实时接收对话模式和分镜模式的助手回复、任务状态与完成事件；轮询只能作为兜底刷新。
- 变更原因：用户要求真正前端实时逐字反馈，需要用浏览器事件流承接后端 planner stream，而不是仅依赖定时轮询数据库。

- 变更类型：新增
- 变更项：对话与分镜 planner 回复流式更新约定
- 变更摘要：新增对话模式和分镜模式的 AI planner 回复必须支持流式更新；任务启动后先创建助手占位消息，并在 planner 流式返回过程中持续更新该消息内容。
- 变更原因：提升对话和分镜模式的即时反馈体验，避免用户必须等待 planner 完整返回后才看到 AI 回复。

- 变更类型：新增
- 变更项：同一会话参考图选择稳定性约定
- 变更摘要：新增同一会话内已选参考图不得因发送对话或分镜消息自动取消；候选参考图片可随新生成图片增加，但不得自动改变已选参考图。
- 变更原因：连续对话和分镜制作需要稳定使用用户明确选中的参考图，避免每次发送后丢失参考约束或被新增候选图自动替换。

- 变更类型：新增
- 变更项：AI planner 与生图执行解耦约定
- 变更摘要：新增对话模式和分镜模式必须支持 planner 与 image_generation 使用不同模型和不同 provider；planner 只负责对话、意图判断和提示词撰写，image_generation 负责实际生图执行。
- 变更原因：支持使用其它渠道模型完成对话与提示词规划，同时保留 GPT 图像链路执行实际生图。

- 变更类型：新增
- 变更项：单张图片提示词约定
- 变更摘要：新增 AI planner 输出的生图提示词必须只对应一张图片；若参考图是用户本轮上传且没有历史生图提示词，禁止编造该参考图的对应提示词。
- 变更原因：避免 planner 将提示词写成说明、流程、多图列表，或错误地为用户上传参考图补造不存在的历史提示词。

- 变更类型：新增
- 变更项：对话与分镜 AI 上下文约定
- 变更摘要：新增对话模式和分镜模式传给 AI planner 的上下文只能包含显式文字消息与已生成图片的文字提示词；选择参考图时必须按当前会话过滤，并将参考图图片与对应提示词/说明成对传入，禁止跨会话串用。
- 变更原因：修复不同任务或不同对话选中参考图混入当前请求、以及隐藏上游上下文导致 AI 误读历史图片关系的问题。

- 变更类型：新增
- 变更项：分镜失败追踪与重试约定
- 变更摘要：新增分镜连续生图失败、取消或意外中断时必须在历史记录中保留镜头状态、已完成图片和失败原因，并支持基于已完成镜头继续重试。
- 变更原因：分镜模式按顺序依赖上一镜头输出图，失败后需要可追踪、可续做，避免用户只能重新开始整组镜头。

- 变更类型：新增
- 变更项：分镜连续生图约定
- 变更摘要：新增分镜连续生图必须先规划镜头列表和一致性概述，再按镜头顺序逐张生成；第 2 个及后续镜头必须以上一镜头输出图作为 edit 输入，并保存中文镜头名与顺序。
- 变更原因：本次新增分镜模式，需要长期约束其不能退化为普通并发批量生图或合图，保证人物、场景和逻辑连续性。

- 变更类型：新增
- 变更项：生图参数约定
- 变更摘要：新增默认电脑壁纸生图参数必须为 `2560x1440` + `high`，禁止将 `1536x1024` 标注为电脑横屏 2K；`1536x1024` 仅作为 3:2 稳定横图展示。
- 变更原因：修正前端“电脑横屏 2K 高清”实际对应 `1536x1024` 导致生成结果不符合电脑壁纸尺寸的问题。

- 变更类型：新增
- 变更项：初始化《项目全局规范》
- 变更摘要：建立项目基础技术栈、目录职责、AI 接口约定、数据存储、默认端口和交付文案的最小统一规则。
- 变更原因：项目根目录原先缺少《项目全局规范》与对应变更记录文件，需要先建立唯一可信的规范基线，保证后续产出统一、可追踪。

- 变更类型：新增
- 变更项：上游默认配置一致性约定
- 变更摘要：新增“前端默认 provider、数据库默认 provider 与后端 `.env` 默认上游必须保持一致，禁止存在相互冲突的默认 `base_url`”规则。
- 变更原因：本次排查发现前后端默认上游来源不一致会直接放大生图链路的不稳定与排查歧义，需要补充为统一约束。

- 变更类型：新增
- 变更项：后端运行环境与重启校验约定
- 变更摘要：新增“后端必须使用 `backend/.venv` 启动，且后端代码、依赖或 `.env` 变更后必须重启进程再验证”的规则。
- 变更原因：本次排查确认运行中的后端进程未使用项目虚拟环境，且磁盘代码版本与常驻进程版本可能脱节，已对生图稳定性与排查准确性造成影响。

- 变更类型：新增
- 变更项：生图 SDK 调用层与容错约定
- 变更摘要：新增“Responses 生图调用层必须优先使用 OpenAI SDK，并支持瞬时失败统一重试与 stream 失败回退非 stream”的规则。
- 变更原因：项目目标是将 `imagegen` skill 的稳定生图逻辑产品化，必须把 SDK、重试和流式回退能力作为长期统一约束。
