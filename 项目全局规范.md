# 项目全局规范

## 1. 规范定位

- 本文件是本项目唯一有效的全局规范文件。
- 后续代码、文档、说明和交付内容必须优先遵守本文件。

## 2. 技术栈与目录职责

- 后端必须使用 FastAPI，后端代码放在 `backend/app`。
- 前端必须使用 React + Vite，前端源码放在 `frontend/src`。
- 生产环境必须由后端托管前端构建产物；前端静态产物目录为 `frontend/dist`。
- 非必要情况下禁止引入 Docker 作为本项目运行前提。

## 3. AI 能力与接口约定

- 普通生图、图片编辑、对话式生图必须优先走 OpenAI 兼容的 `/v1/responses` 接口。
- 图片生成能力必须基于 `image_generation` 工具实现。
- `/v1/responses` 生图调用层必须优先使用 OpenAI SDK，不得重新维护裸 `httpx` 生图请求实现。
- 生图调用层必须支持对 `429/502/503/504/upstream_error/timeout` 等瞬时失败的统一重试，并支持 stream 失败后自动回退到非 stream 请求。
- 对话式改图仅在用户显式上传或选择参考图时允许进入编辑流程，禁止自动猜测历史图片作为编辑输入。
- 对话模式参考图总数必须限制为最多 3 张。
- 对话模式和分镜模式传给 AI planner 的上下文必须只包含显式文字消息与已生成图片对应的文字提示词，禁止依赖隐藏的上游 `previous_response_id` 链作为上下文。
- 用户选择参考图时，后端必须按当前会话过滤；发送给 AI planner 时必须将参考图图片与其对应提示词/说明成对传入，并明确标识二者关系。
- 不同会话或不同任务中选中的参考图禁止相互串用；前端切换会话时必须清理或过滤当前已选参考图。
- 同一会话内用户已选参考图必须保持稳定，禁止因发送对话或分镜消息自动取消；仅允许用户手动取消、切换会话、新建会话或删除会话时清理/过滤。
- 同一会话内候选参考图片列表必须随新生成图片增加而更新，但候选列表变化不得自动改变已选参考图。
- 对话模式和分镜模式必须支持 AI planner 与 image_generation 使用不同模型和不同 provider；planner 负责对话、意图判断和提示词撰写，image_generation 负责实际生图执行。
- 最终生图执行必须使用全局生图提供商池；普通生图、编辑、对话式生图、分镜生图与分镜重试都必须从同一生图池中选择最终执行 provider。
- 生图并发上限必须按“单提供商”维度限制，默认每个 provider 最多同时运行 3 个生图任务；项目总并发容量应等于“池内 provider 数 × 单 provider 上限”。
- 生图池调度必须优先选择当前没有运行生图任务的 provider；若池内 provider 都在运行任务，则必须优先选择运行中任务数最少的 provider。
- 任务记录、任务详情和前端运行态界面必须展示当前实际使用的生图 provider；前端提供商管理界面必须展示生图池总 provider 数、已使用 provider 数和空闲 provider 数。
- AI planner 接口格式必须默认使用 `/v1/responses`；仅在用户显式选择时允许切换为 `/v1/chat/completions`。`chat/completions` 兼容仅限 planner 文本规划，不得影响 image_generation 生图执行链路。
- AI planner 使用 `chat/completions` 兼容模式时，只保证传入文字对话、参考图文字说明和已知生图提示词；若需要 planner 直接读取参考图图片本体，必须使用默认 `responses` 模式。
- 对话模式和分镜模式的 AI planner 回复必须支持流式更新；任务启动后应先创建助手占位消息，并在 planner 流式返回过程中持续更新该消息内容。
- 前端必须通过任务级 SSE 事件流实时接收对话模式和分镜模式的助手回复、任务状态与完成事件；轮询只能作为兜底刷新，禁止作为唯一实时反馈机制。
- 对话模式与分镜模式必须使用各自独立的会话，禁止在同一会话内混用不同模式。
- 普通生图模式和图片编辑模式也必须按模式归属到各自独立会话；同一会话内允许连续提交多次生图或编辑请求，但每次都必须把用户当次输入视为完整独立提示词，禁止自动拼接历史提示词、禁止调用 planner 生成助手回复、禁止对用户提示词做 AI 扩写后再执行。
- 同一会话存在 `queued` 或 `running` 状态任务时，禁止继续向该会话发送新消息；仅允许先停止该会话任务，或新建会话后继续。
- 从历史记录继续某个对话时，必须复用原 `conversation_id` 与原会话上下文，禁止另起新会话冒充“继续对话”。
- 对话模式和分镜模式任务失败后，继续原会话时传给 AI planner 的上下文只允许包含用户消息、已完成的助手回复和已成功生成图片对应的提示词；失败原因、未完成助手占位内容和失败生图提示词禁止进入后续上下文。
- AI planner 输出的生图提示词必须明确只对应一张图片；若参考图是用户本轮上传图片且没有历史生图提示词，禁止为该参考图编造对应提示词。
- 分镜连续生图必须先由 AI 规划镜头列表和人物/场景一致性概述，再按镜头顺序逐张生成；第 2 个及后续镜头必须以上一镜头输出图作为 edit 输入，禁止改成一次性合图或普通并发批量生图。
- 分镜连续生图的每张输出必须对应一个独立镜头，并保存中文镜头名与镜头顺序。
- 分镜连续生图每完成并保存一张镜头图片后，后端必须立即通过任务级 SSE 推送该镜头状态和图片信息，前端必须立即展示该图片，禁止等全部镜头完成后才统一刷新。
- 分镜连续生图失败、取消或意外中断时，历史记录必须保留已规划镜头、每个镜头状态、已完成图片和失败原因；历史详情必须支持基于已完成镜头继续重试。
- 普通生图、图片编辑和分镜任务在失败或取消后，必须支持基于原请求参数、原输入图和原会话约束发起重试；若该任务属于会话任务，重试时同样必须遵守“同一会话同一时刻只能有一个运行中或排队中任务”。

## 4. 数据存储约定

- 本项目默认持久化存储必须使用 SQLite。
- 默认数据库文件位置必须为 `storage/app.db`，除非环境变量明确覆盖。
- 上传图片必须保存到 `storage/uploads`，生成结果必须保存到 `storage/outputs`。
- 提供商、应用设置、任务、会话、消息、提示词和图片记录必须优先落到 SQLite，不依赖浏览器本地存储作为唯一数据来源。
- 项目更新脚本禁止覆盖、重置或重写云服务器已有 SQLite 业务数据，包括 provider、settings、tasks、conversations、messages、prompts、images 等表内记录。
- 若版本更新涉及 SQLite 表结构变化，更新脚本必须只执行向前兼容的 schema 迁移，例如创建缺失表或补充缺失列，禁止借迁移覆盖已有业务记录。

## 5. 运行与配置约定

- 默认服务端口必须为 `8010`，除非环境变量 `PORT` 明确覆盖。
- 与上游图片服务相关的默认配置必须通过 `.env` 或后端配置统一读取，禁止把密钥硬编码到后端逻辑中。
- 前端默认 provider、数据库中的默认 provider 与后端 `.env` 默认上游必须保持一致，禁止同时存在相互冲突的默认 `base_url`。
- 启动后端服务时必须优先使用项目内 `backend/.venv` 的 Python 解释器，禁止使用系统 Python、Anaconda 或其它项目外解释器直接启动本项目后端。
- 后端代码、后端依赖或 `.env` 配置发生变更后，若服务进程常驻运行，必须重启对应后端进程后再验证结果，禁止把未重启进程的运行结果视为最新代码行为。
- 任务执行必须支持后台运行；页面刷新或切换后不得丢失任务状态。
- 项目访问必须先通过统一密码页验证；未完成验证前，禁止直接访问首页、前端静态资源、媒体资源和业务 API。

## 6. 生图参数约定

- 默认电脑壁纸生图参数必须为 `2560x1440` + `high`，禁止将 `1536x1024` 标注为电脑横屏 2K。
- `1536x1024` 仅允许作为 3:2 稳定横图选项展示。

## 7. 文案与交付约定

- 面向用户的默认产品文案、提示信息和错误说明应优先使用中文。
- Git 提交信息必须使用中文；提交标题必须为中文短标题，提交正文必须使用中文详细描述本次改动的核心内容与目的。
- 描述项目能力时，应基于“对话理解、生图执行、历史沉淀、任务可追踪”四类核心能力统一表述，避免与实际实现不一致的夸大描述。
- 前端图片预览必须使用站内固定大弹窗，禁止默认打开新页面；预览框必须支持原图展示、拖拽移动、缩放，以及在同一任务、会话或图库分组内上一张/下一张切换。
- 历史记录页必须展示每条记录所属的生图模式，并支持按模式筛选历史记录列表。
- 图库页必须按会话聚合同一会话的全部历史图片；当单个会话图片过多时，默认只展示部分预览图，并提供“查看详情”入口展开该会话全部图片。
- 图库页必须展示每个会话图片分组所属的生图模式，并支持按模式筛选会话图片分组。
- 从历史记录或图库点击“继续改”时，若目标图片不属于当前会话同模式的可复用生成图，前端必须将该图片转换为本轮上传参考图再进入处理界面，禁止仅保存图片 id 后依赖当前会话过滤去猜测是否可用。
- 对话/分镜工作台底部对“已选参考图”必须展示真实缩略图预览，禁止只显示已选数量或文字提示而不展示具体图片。
- 设置面板中的每个独立设置分组必须提供单独的“保存修改”按钮；点击后必须在该按钮上方原位显示成功或失败反馈，禁止只做静默自动保存或仅通过全局错误面板提示结果。
